常见问题说明

1. 我该如何反馈问题？

  • 可以在 iCost 客户端的设置里面，找到反馈按钮，点击之后可以直接提交反馈；
  • 或者你可以直接发送邮件到 gostraight_tech@outlook.com，我们会及时回复您的邮件；
  • 加入我们的反馈 QQ 群 665434194，有问题可以随时进行反馈，我们也同样会尽快进行回复;
  • 加入我们的 Telegram 群，https://t.me/iCostTG。

2. 资产账户余额更新说明

添加支出、收入、转账记录时，选择对应的账户，即可更新余额。

2.1 修改账户余额后又补记了支出收入记录，会计算入账户余额吗？

如果修改了账户余额，会以修改的时间点为基准进行计算。

会出现以下两种情况：

  • 补记的账单所选择的时间，如果在账户余额修改的时间点之后，则会在修改之后的账户余额上进行叠加计算；
  • 补记的账单所选择的时间，如果在账户余额修改的时间点之前，则账户余额不会发生改变。

2.2 手动修改账户余额后，为什么账户里会出现一条修改记录？可以删除吗？

修改记录的存在是为了更好的了解账户余额的变更情况。当出现补记账单之后，账户余额没有更新的时候，可以快速定位，确保资产账户始终保持正确的余额。

单条修改记录可以左滑删除，但是删除之后，账户余额会以上一条修改记录的金额为基准进行计算。如果删除的是账户创建所产生的第一条修改记录，则删除之后会以 0 为基准进行计算。

2.3 添加资产账户后，记录以前的支出收入，为什么不更新账户余额呢？

添加资产账户时，会以填写的金额作为当前的基准进行计算。记录以前的支出收入，是一种补记行为，不会影响当前的账户余额。具体逻辑，参考 3.1 的情况。

3. 小组件不更新金额或者黑屏的解决办法

如果更新应用后出现小组件金额显示为 0，或者直接出现黑屏的情况，可以通过新增一笔记账记录或者重启手机系统的方式尝试解决。

针对现有用户的反馈，目前大部分的小组件显示问题，都可以通过主动触发刷新机制来解决。这可能是系统没有及时刷新小组件的内容所导致的问题，我们也会在后续的版本中，增加小组件的主动刷新次数来减少或者避免这类事情的出现。

4. 应用是否存在账户系统？

没有账户系统。

需要再次申明的一点是，iCost 绝不存储和使用您的数据。可能有的用户会疑惑，在开发者没有保存用户记账数据的同时，为什么会需要添加头像和昵称呢？

这里有 3 个原因：

  1. 为后续构建成就系统做准备；
  2. 希望和用户建立一种联系，从细节提高用户的使用体验；
  3. 重构图表，让统计成为应用第三大功能的同时，弱化设置功能对用户注意力的吸引。（2021/03/07 版本 v3.4.0）。

5. 如果 Apple Watch 应用出现没有分类/账本/货币怎么办？

Apple Watch 的数据是从 iPhone 中实时传输过去的。一般等待1-2秒即可。

如果出现数据长时间不出现的问题，可以选择重启一下 iPhone 设备，此时会重新建立起两端设备之间的一个联系。
